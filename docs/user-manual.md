## 第一章 简介

### 1.1 什么是 Life OS？

Life OS 是一个**个人人生管理工具**，旨在帮助你系统化地管理生活中的方方面面。它借鉴了操作系统的工作原理，将任务管理、项目规划、目标追踪、日记记录、决策分析等功能整合在一个统一的界面中。

与传统的笔记软件或待办应用不同，Life OS 采用了**纯文件系统**的数据存储方式。你的所有数据都以 Markdown 格式存储在本地，你可以使用任何编辑器查看和编辑它们。

### 1.2 核心设计理念

**数据本地化**
- 所有数据存储在你的电脑上，不依赖云服务
- 没有隐私泄露风险
- 完全掌控自己的数据

**Markdown 格式**
- 纯文本存储，可读性强
- 支持富文本渲染
- 可被任何编辑器打开

**文件系统为根**
- Vault（知识库）就是一个普通的文件夹
- 可以使用文件管理器浏览
- 可以使用 Git 进行版本控制

### 1.3 与传统工具的对比

| 特性 | Life OS | Notion | Obsidian |
|------|---------|--------|----------|
| 数据存储 | 本地文件系统 | 云端 | 本地（可同步） |
| 数据格式 | Markdown | 专有 + Markdown | Markdown |
| 定制能力 | 高度可定制 | 有限 | 有限 |
| 离线使用 | 完全支持 | 部分支持 | 完全支持 |
| 版本控制 | 原生支持 | 不支持 | 需额外配置 |
| 跨平台 | 仅 Mac | 全部平台 | 全部平台 |

### 1.4 适用场景

Life OS 特别适合以下人群：

- **效率爱好者**：希望有一个统一的地方管理所有事务
- **开发者**：喜欢 Markdown 和纯文本工作流
- **隐私敏感者**：不想把个人信息交给云服务
- **自驱力强的人**：需要一个系统来追踪目标和习惯
- **决策者**：需要记录和分析重要决策

## 第二章 快速开始

### 2.1 系统要求

- **操作系统**：macOS 11.0 或更高版本
- **处理器**：Intel 或 Apple Silicon
- **内存**：至少 4GB RAM（推荐 8GB）
- **硬盘**：至少 500MB 可用空间

### 2.2 前置软件安装

#### 安装 Rust

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

#### 安装 Xcode Command Line Tools

```bash
xcode-select --install
```

#### 安装 Node.js（推荐使用 nvm）

```bash
# 如果还没有安装 nvm
brew install nvm

# 安装并使用 Node.js 20
nvm install 20
nvm use 20
```

### 2.3 安装 Life OS

```bash
# 进入项目目录
cd life-os

# 安装前端依赖
npm install

# 启动开发版本
npm run tauri dev
```

> **注意**：首次运行需要编译 Rust 后端，大约需要 3-5 分钟。后续启动会快很多。

### 2.4 首次启动和 Vault 设置

首次启动 Life OS 时，你会看到一个设置界面，需要指定 Vault（知识库）的位置。

**Vault 是什么？**
Vault 是存储所有数据的文件夹。你可以：

1. **选择现有文件夹**：如果你已经有组织好的数据
2. **创建新文件夹**：让 Life OS 为你创建一个新的 Vault

**推荐做法**：
- 将 Vault 放在 iCloud Drive 文件夹中，实现自动云同步
- 或者将 Vault 初始化为 Git 仓库，获得版本历史

### 2.5 创建你的第一条任务

首次设置完成后，你会看到仪表盘页面。尝试以下操作：

1. 在**今日任务**区域添加你的第一个任务
2. 在**习惯打卡**区域点击一个习惯完成打卡
3. 浏览各个功能模块，熟悉界面布局

## 第三章 核心概念

### 3.1 Vault 文件结构

```
~/life-os-vault/
├── .life-os/
│   ├── config.yaml          # 全局配置
│   └── connectors.yaml      # API tokens（不提交到 git）
├── daily/
│   ├── tasks/
│   │   └── 2025-02-19.md   # 每天一个任务文件
│   └── habits/
│       └── habits.yaml      # 习惯定义 + 打卡记录
├── projects/
│   ├── active/              # 进行中项目
│   ├── backlog/            # 待规划项目
│   ├── todo/               # 计划中项目
│   └── done/               # 已完成项目
├── planning/
│   └── goals/              # 目标文件（年/季/月）
├── diary/
│   └── 2025/               # 按年份分目录
├── decisions/              # 决策记录
└── connectors/              # 外部数据缓存
```

### 3.2 Markdown + YAML Frontmatter

Life OS 使用 Markdown 格式存储内容，并在文件顶部使用 YAML frontmatter 存储元数据。

**任务文件示例**（`daily/tasks/2025-02-19.md`）：

```markdown
---
date: 2025-02-19
energy: high
mood: 😊
---

## 今日任务

- [ ] 晨间冥想 #habit #health
- [x] 健身 45分钟 #health ⏰09:30
- [ ] 阅读 30分钟 #learning

## 今日笔记

今天完成了重要的项目汇报...
```

**项目文件示例**（`projects/active/ai-saas-mvp.md`）：

```markdown
---
title: AI SaaS MVP
status: active
priority: high
created: 2025-01-15
updated: 2025-02-19
due: 2025-06-01
tags: tech, startup
progress: 68
github: username/repo
---

## 目标

打造一个面向小企业的 AI 助手产品...

## 任务

- [x] 完成用户调研
- [ ] 设计产品原型
- [ ] 开发核心功能
```

### 3.3 标签系统

使用 `#tag` 语法为任务、笔记等内容添加标签。

**常用标签约定**：

| 标签 | 含义 | 示例 |
|------|------|------|
| `#habit` | 习惯养成 | `#habit #health` |
| `#health` | 健康相关 | `#health` |
| `#learning` | 学习 | `#learning` |
| `#work` | 工作 | `#work` |
| `#project-name` | 特定项目 | `#ai-saas` |

### 3.4 时间标记

使用 `⏰HH:MM` 语法为任务设置提醒时间：

```markdown
- [ ] 晨间会议 ⏰09:30
- [ ] 健身 ⏰18:00
- [ ] 阅读 ⏰21:00
```

### 3.5 任务状态

| 状态 | 符号 | 含义 |
|------|------|------|
| 未完成 | `[ ]` | 待办任务 |
| 已完成 | `[x]` | 已完成任务 |

## 第四章 功能模块使用指南

### 4.1 仪表盘（Dashboard）

仪表盘是 Life OS 的首页，汇集了你所有的关键信息。

**主要功能：**

- **今日概览**：显示当前日期、星期、问候语
- **任务统计**：待完成任务数、已完成数、完成率
- **项目概览**：进行中的项目数量
- **习惯打卡**：显示今日习惯和完成状态
- **最近日记**：快速访问最新日记

**使用技巧：**

1. 点击任何统计卡片可以跳转到对应的功能页面
2. 习惯打卡后会有视觉反馈（绿色边框和勾选标记）
3. 任务完成率会影响激励语的内容

### 4.2 每日任务（DailyView）

每日任务模块帮助你管理每一天的具体工作。

**核心功能：**

- **添加任务**：在输入框中输入任务，按 Enter 确认
- **完成任务**：点击复选框标记为完成
- **设置提醒**：使用 `⏰时间` 语法
- **添加标签**：使用 `#标签` 语法
- **删除任务**：已完成的任务可以删除

**使用格式：**

```markdown
- [ ] 任务描述 #标签 ⏰时间
- [x] 已完成任务 #已完成
```

**历史记录：**

- 使用日历视图查看历史任务
- 点击任意日期可以查看当天的任务记录
- 只读模式查看，不能修改历史数据

### 4.3 习惯养成（Habits）

习惯模块帮助你追踪和养成好习惯。

**配置习惯：**

习惯定义存储在 `daily/habits/habits.yaml` 文件中：

```yaml
habits:
  - id: meditation
    name: "晨间冥想"
    icon: "🧘"
    target_days: [1,2,3,4,5,6,7]
    created: "2025-01-01"
```

**打卡操作：**

- 在仪表盘或每日任务页面点击习惯卡片完成打卡
- 打卡后会显示绿色对勾标记
- 可以追踪每日/每周/每月的完成率

### 4.4 项目看板（KanbanView）

项目看板采用四阶段管理方式，帮助你可视化项目的进展。

**四个阶段：**

| 阶段 | 标签 | 含义 |
|------|------|------|
| 待规划 | 💤 | 想法收集阶段 |
| 计划中 | 📋 | 正在制定计划 |
| 进行中 | ⚡ | 积极执行中 |
| 已完成 | ✅ | 已交付完成 |

**操作方式：**

1. **创建项目**：在任何列中点击"+ 添加"按钮
2. **移动项目**：拖拽项目卡片到其他列
3. **编辑项目**：点击卡片查看详细信息
4. **设置优先级**：高/中/低三级

**优先级显示：**

| 优先级 | 颜色 | 标签 |
|--------|------|------|
| 高 | 紫色 | ↑高 |
| 中 | 橙色 | →中 |
| 低 | 灰色 | ↓低 |

### 4.5 目标计划（PlanningView）

目标计划模块帮助你制定和追踪长期目标。

**目标类型：**

- **年度目标**：一整年的愿景和目标
- **季度目标**：三个月的阶段性目标
- **月度目标**：当月的具体行动计划

**创建目标：**

1. 点击"新建目标"按钮
2. 选择目标类型（年/季/月）
3. 填写目标标题和描述
4. 设置截止日期和优先级
5. 添加里程碑（可选）

**进度追踪：**

- 使用滑块更新目标进度（0-100%）
- 里程碑显示为勾选框
- 支持添加标签进行分类

### 4.6 日记（DiaryView）

日记模块记录你的日常想法和感受。

**核心功能：**

- **心情记录**：选择当天的情绪表情
- **自由写作**：Markdown 格式的日记编辑
- **历史浏览**：按日期查看历史日记
- **标签分类**：使用标签组织日记

**新建日记：**

1. 点击右上角的新建按钮
2. 选择日期（默认为今天）
3. 选择心情
4. 开始写作

**心情选项：**

😊 开心 | 😐 普通 | 😔 失落 | 😤 生气
🤔 思考 | 😴 疲惫 | 🤩 兴奋 | 😢 难过

### 4.7 决策记录（DecisionsView）

决策模块帮助你记录和分析重要的人生决策。

**使用场景：**

- 职业选择
- 投资理财
- 生活重大决定
- 任何需要权衡利弊的场景

**记录决策：**

1. 点击"新建决策"
2. 填写决策标题
3. 描述决策背景
4. 列出支持理由
5. 列出反对理由
6. 设置重要程度

**重要程度：**

| 等级 | 含义 |
|------|------|
| 低 | 日常小决定 |
| 中 | 需要考虑的决定 |
| 高 | 重要人生选择 |
| 关键 | 影响人生的重大决定 |

**AI 辅助分析：**

如果你的电脑上安装了 Claude CLI，可以启用 AI 辅助决策分析。AI 会：

- 总结决策背景
- 分析支持/反对理由
- 提供决策建议
- 建议复盘时间

### 4.8 便利贴（StickyNotesView）

便利贴模块提供了一个无限画布，让你可以像在真实墙面上一样粘贴便签。

**创建便签：**

1. 在画布空白处双击
2. 选择便签颜色
3. 输入内容

**便签颜色：**

| 颜色 | 含义 |
|------|------|
| 黄色 | 通用/提醒 |
| 粉色 | 个人/情感 |
| 蓝色 | 工作/学习 |
| 绿色 | 健康/习惯 |
| 橙色 | 紧急/重要 |

**操作方式：**

- **拖拽移动**：点击便签主体拖动
- **调整大小**：拖拽右下角调整大小
- **旋转**：便签会有轻微随机旋转
- **一键排版**：点击工具栏的网格按钮自动对齐

### 4.9 定时任务（SchedulerView）

定时任务模块可以管理系统级的定时任务（仅限 macOS）。

**创建定时任务：**

1. 点击"新建任务"
2. 填写任务名称
3. 选择执行的程序
4. 设置参数（可选）
5. 设置执行间隔

**预设模板：**

- **每日备份**：每天凌晨 3 点执行
- **每小时提醒**：每小时提醒喝水
- **每周清理**：每周日清理下载文件夹

### 4.10 Git 仓库扫描器（GitScannerView）

Git 扫描器帮助你管理本地所有的 Git 仓库。

**扫描仓库：**

1. 选择要扫描的根目录
2. 设置扫描深度（推荐 2-3 层）
3. 点击扫描按钮

**功能特性：**

- 显示仓库名称和当前分支
- 标记有未提交改动的仓库
- 快速在 Finder 中打开仓库
- 复制仓库路径

### 4.11 Skills 管理器（SkillsView）

Skills 管理器帮助你组织 AI 助手（如 Claude Code、Cursor、VS Code）的技能文件。

**支持的 IDE：**

- Claude Code
- Cursor
- VS Code（Cline）

**操作方式：**

1. 选择要管理的 IDE
2. 查看现有 Skills 列表
3. 点击编辑查看/修改技能内容
4. 创建新的 Skills

### 4.12 连接器（ConnectorsView）

连接器模块集成了第三方服务。

**GitHub 连接：**

1. 在 GitHub 生成 Personal Access Token
2. 在设置页面添加 Token
3. 连接后可以：
   - 查看用户信息
   - 同步仓库列表
   - 查看通知

### 4.13 设置（Settings）

设置页面管理 Life OS 的全局配置。

**功能选项：**

- **Vault 路径**：更改数据存储位置
- **主题切换**：深色/浅色模式
- **数据迁移**：将数据从一个 Vault 迁移到另一个

**主题：**

| 主题 | 背景色 | 适用场景 |
|------|--------|----------|
| 深色 | 深蓝黑 | 夜间使用 |
| 浅色 | 灰白色 | 白天使用 |

## 第五章 数据管理

### 5.1 Vault 备份策略

**推荐方案：**

1. **iCloud Drive 同步**
   - 将 Vault 放在 iCloud Drive 文件夹中
   - 自动云同步
   - 跨设备访问

2. **Git 版本控制**
   ```bash
   cd ~/life-os-vault
   git init
   git add .
   git commit -m "Initial commit"
   ```

### 5.2 手动备份

定期备份 Vault：

```bash
# 复制整个文件夹
cp -r ~/life-os-vault ~/life-os-vault-backup-$(date +%Y%m%d)
```

### 5.3 数据迁移

迁移 Vault 到新位置：

1. 打开设置页面
2. 点击"更换 Vault 路径"
3. 选择新位置
4. 选择：
   - **迁移**：复制现有数据到新位置
   - **新建**：清空数据使用新 Vault

### 5.4 跨设备同步

如果需要跨设备使用：

1. 将 Vault 放在 iCloud Drive、Dropbox 或其他同步服务
2. 在每个设备上设置相同的 Vault 路径
3. 注意：同一时间只能在一个设备上编辑

## 第七章 常见问题

### 7.1 安装问题

**Q: 首次运行很慢？**
A: 首次运行需要编译 Rust 后端，需要 3-5 分钟。后续启动会快很多。

**Q: 编译失败怎么办？**
A:
1. 确保已安装 Xcode Command Line Tools
2. 确保 Rust 已正确安装
3. 运行 `rustc --version` 验证

### 7.2 路径配置问题

**Q: 如何更改 Vault 路径？**
A: 打开设置页面，点击"更换 Vault 路径"，选择新位置。

**Q: Vault 丢失怎么办？**
A:
1. 如果使用 iCloud Drive，检查 iCloud 同步状态
2. 如果使用 Git，运行 `git restore .` 恢复

### 7.3 数据同步问题

**Q: 多个设备同时编辑？**
A: 不建议。可能会导致冲突。建议使用 Git 分支或手动合并。

**Q: 数据没有保存？**
A: Life OS 会实时保存到文件系统。检查 Vault 路径设置是否正确。

### 7.4 功能使用问题

**Q: 如何删除任务？**
A: 在每日任务视图中，已完成的任务右侧有删除按钮。

**Q: 便签不见了？**
A: 使用画布的缩放和拖动功能查看整个画布，或点击"重置视图"。

**Q: GitHub 同步失败？**
A: 检查 Token 是否有效，GitHub API 是否有访问限制。

### 7.5 性能优化

**Q: 应用变慢了？**
A:
1. 减少同时打开的功能模块数量
2. 定期清理已完成的项目
3. 归档旧的日记和决策
