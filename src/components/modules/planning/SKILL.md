# 目标计划技能

此技能用于通过 AI 辅助管理年度、季度、月度目标，支持目标创建、进度追踪和完成度分析。

## 触发条件

- 用户输入 `/目标` 或 `/计划`
- 用户提及 "目标"、"计划"、"年度目标"、"季度目标"
- 用户要求设置或追踪目标进度

## 可用命令

### 创建目标

```
/目标 新建 [标题] [类型]
/计划 创建 [年度/季度/月度] 目标 [标题]
```

示例：
```
/目标 新建 "学习英语" 年度
/计划 创建季度目标 "完成项目A"
/目标 新建 "健身" 月度
```

### 查看目标

```
/目标 列表
/目标 查看 [类型]
/目标 进度 [目标标题]
```

示例：
```
/目标 列表
/目标 查看年度目标
/目标 进度 "学习英语"
```

### 更新进度

```
/目标 更新 [目标标题] 进度 [百分比]
/目标 完成 [目标标题]
```

示例：
```
/目标 更新 "学习英语" 进度 50%
/目标 完成 "健身"
```

### 编辑目标

```
/目标 编辑 [目标标题] [属性] [值]
```

示例：
```
/目标 编辑 "学习英语" 优先级 高
/目标 编辑 "完成项目A" 截止日期 2025-03-31
```

### 删除目标

```
/目标 删除 [目标标题]
```

### 目标分析

```
/目标 分析
/目标 报告 [类型]
/目标 总结
```

## 数据存储

### 目标文件路径

```
{vault}/planning/goals/{year}-{type}-{slug}.md
```

### 文件命名规则

- 年度目标: `{year}-annual-{slug}.md`
- 季度目标: `{year}-q{quarter}-{slug}.md`
- 月度目标: `{year}-mm{month}-{slug}.md`

### Frontmatter 格式（目标）

```yaml
---
title: 学习英语
type: annual
year: "2025"
quarter: "1"
month: "1"
status: active
progress: "50"
priority: high
due: "2025-12-31"
tags: 学习,自我提升
created: "2025-01-01"
updated: "2025-01-15"
---
```

### 目标内容模板

```markdown
## 目标说明

学习英语，提升听说读写能力

## 关键结果

- 通过雅思 6.5 分
- 每周背诵 100 个单词
- 每月看 2 部英文电影

## 执行计划

- 每天早上背单词 30 分钟
- 每周参加英语角活动
- 每月进行一次模拟考试
```

## 目标类型

### 年度目标
- 时间跨度：1 年
- 建议数量：3-5 个
- 检查频率：每季度
- 适用场景：长期规划、年度愿景

### 季度目标
- 时间跨度：3 个月
- 建议数量：2-4 个/季度
- 检查频率：每月
- 适用场景：中期计划、项目里程碑

### 月度目标
- 时间跨度：1 个月
- 建议数量：3-5 个/月
- 检查频率：每周
- 适用场景：短期目标、习惯养成

## 优先级说明

- **紧急 (urgent)**: 需要立即处理
- **高 (high)**: 优先处理
- **中 (medium)**: 正常处理
- **低 (low)**: 有空再处理

## 状态说明

- **进行中 (active)**: 正在执行
- **已完成 (completed)**: 已达成
- **已归档 (archived)**: 不再追踪

## AI 操作指南

### 读取目标列表

使用 `listNotes("{vault}/planning/goals", false)` 读取所有目标文件，解析 frontmatter 获取目标信息。

### 创建目标

使用 `writeNote()` 写入新的目标文件，文件名遵循命名规则，frontmatter 包含目标属性。

### 更新进度

读取目标文件，修改 frontmatter 中的 progress 字段，然后写回文件。

### 搜索目标

使用 listNotes 结合关键词搜索，或按 frontmatter 字段过滤（如 type、status、priority）。

## 目标健康度评估

### 优秀 (90-100)
- 活跃目标有进度更新
- 完成率 > 20%
- 目标分布合理

### 良好 (70-89)
- 大部分目标有进度
- 有少量已完成目标
- 目标类型较平衡

### 一般 (50-69)
- 部分目标长期未更新
- 完成率较低
- 目标类型单一

### 较差 (30-49)
- 大部分目标停滞
- 无已完成目标
- 缺乏定期回顾

### 危险 (0-29)
- 目标完全被忽视
- 无进度追踪
- 需要重新规划

## 注意事项

1. 目标文件使用 Markdown + frontmatter 格式
2. 目标 slug 自动生成，避免中文和特殊字符
3. 进度更新会自动修改 updated 字段
4. 删除目标需要确认操作
5. 建议定期回顾和调整目标
