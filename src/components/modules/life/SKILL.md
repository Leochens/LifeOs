# 生活数据技能

此技能用于追踪和管理个人生活数据，包括屏幕使用时间和健康指标，帮助平衡数字生活与身体健康。

## 触发条件

- 用户输入 `/生活` 或 `/健康`
- 用户提及 "屏幕时间"、"使用时间"、"步数"、"睡眠"
- 用户想要查看或分析生活数据趋势

## 可用命令

### 刷新数据

```
/生活 刷新
/生活 更新数据
```

### 查看数据

```
/生活 屏幕时间
/生活 健康数据
/生活 今日概览
```

### 数据分析

```
/生活 分析本周屏幕时间
/生活 健康趋势
/生活 使用时长排行
```

### 设置目标

```
/生活 设置屏幕时间目标 [小时]
/生活 设置步数目标 [步数]
```

## 屏幕使用时间指标

### 总使用时间
- 🟢 健康：< 4 小时/天
- 🟡 一般：4-6 小时/天
- 🔴 过度：> 6 小时/天

### 拿起次数
- 🟢 健康：< 50 次/天
- 🟡 一般：50-80 次/天
- 🔴 过度：> 80 次/天

### 使用分类建议
- 工作/学习：建议占比 > 40%
- 社交：建议占比 < 30%
- 娱乐：建议占比 < 20%

## 健康数据指标

### 步数
- 🟢 优秀：> 10,000 步/天
- 🟡 良好：5,000-10,000 步/天
- 🔴 不足：< 5,000 步/天

### 活动分钟
- 🟢 优秀：> 60 分钟/天
- 🟡 良好：30-60 分钟/天
- 🔴 不足：< 30 分钟/天

### 睡眠
- 🟢 优秀：7-9 小时/天
- 🟡 一般：6-7 小时或 9-10 小时
- 🔴 不足：< 6 小时或 > 10 小时

### 心率
- 静息心率正常范围：60-100 BPM
- 运动员可能更低：40-60 BPM

## 数据来源

### 屏幕使用时间 (Screen Time)

通过 macOS Shortcuts 获取，需创建 "Get Screen Time" 捷径：

```json
{
  "totalMinutes": 360,
  "byCategory": {
    "社交": 120,
    "工作": 180,
    "娱乐": 60
  },
  "byApp": {
    "Slack": 90,
    "Chrome": 120,
    "VS Code": 60
  },
  "pickupCount": 45
}
```

### 健康数据 (Health)

通过 macOS Shortcuts 获取，需创建 "Get Health Data" 捷径：

```json
{
  "steps": 8500,
  "activeMinutes": 45,
  "calories": 2100,
  "sleepHours": 7.5,
  "heartRate": 72
}
```

## AI 操作指南

### 获取屏幕时间数据

使用 `runShortcut("Get Screen Time")` 获取屏幕使用时间：

```typescript
const stResult = await runShortcut("Get Screen Time");
const st = JSON.parse(stResult);
```

### 获取健康数据

使用 `runShortcut("Get Health Data")` 获取健康数据：

```typescript
const healthResult = await runShortcut("Get Health Data");
const hd = JSON.parse(healthResult);
```

### 降级处理

如果 Shortcuts 不可用，使用模拟数据作为降级方案。

## 综合健康评估

健康的生活状态应满足：
- 平衡工作与休息时间
- 保持适量日常运动
- 控制屏幕使用时间
- 确保充足睡眠

## 注意事项

1. 首次使用需要在系统设置中授予屏幕使用时间和健康数据的访问权限
2. 需要在 Shortcuts 应用中创建对应捷径才能获取实时数据
3. 如未配置捷径，将显示模拟数据
4. 数据每日更新，建议每天固定时间查看
